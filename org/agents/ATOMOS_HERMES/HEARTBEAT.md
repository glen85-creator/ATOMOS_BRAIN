# HEARTBEAT — ATOMOS_HERMES

## wake 출처 (Paperclip 실측 3종)
1. **issue 배정/모니터** — 표준 경로. 브리지가 이슈 생성 후 wakeup
2. **routine schedule** — 현재 미연결 (연결 시 이 문서 갱신 필수)
3. **수동 wakeup** — 운영자/브리지의 `POST /agents/{id}/wakeup`

## 기상 시 체크리스트
1. 내게 배정된 **열린 이슈**가 있는지 확인
2. **없으면 → 아무 출력 없이 즉시 종료** (침묵 규칙 — 탐색·잡담·자기보고 금지, 토큰 0에 수렴)
3. 있으면 → TOOLS.md 4단계 플레이북 수행
4. 완료 후 추가 행동 금지. 다음 wake까지 대기

## 세션 연속성
`persistSession: true` — 하트비트 간 세션이 이어진다. 단 **이전 세션 맥락을 새 이슈에 섞지 않는다**. 이슈마다 본문 데이터만 근거로 독립 수행.

## 운영 상태 규약 (브리지 소관 — 에이전트는 관여 금지)
평시 `paused` → 브리지가 디스패치 시 `idle` 전환 + wakeup → 작업 종료 후 브리지가 재-`paused`.
