# ATOMOS 파이프라인 재설계 — 설계 문서 (LIVING / 진행 중)

> 상태: **브레인스토밍 진행 중** — 결정된 것 + 미결을 계속 갱신하는 살아있는 문서.
> 작성 시작: 2026-06-20. 대화로 합의된 결정을 누적 기록.

## 0. 배경 / 동기
- 기존 파이프라인(감지→외주→보고)이 개별 버그 땜질로는 해결 안 됨이 드러남(중복 차단·합성 flaky·발송 게이트 양방향 버그·토큰 분산·정량화 부족 등).
- **Paperclip(보드)·hermes(에이전트)는 Hostinger 관리형 이미지** = 우리 소스 아님, 휘발 패치만 가능한 블랙박스 → 의존이 문제의 근원.
- 결정: **전체 파이프라인을 우리가 통제하는 구조로 재설계.**

## 1. 범위 / 척추 결정 (확정)
- **범위**: 전체 파이프라인 재설계(감지→분석→승인→실행→측정→보고).
- **실행 모델**: **역할 단일단계** — 도메인별 역할 1개가 분석(다중 에이전트 팬아웃·CEO 합성 제거).
- **"실행" 정의**: **사람 승인 → 한정 안전도구 자동실행** + 효과 측정.
- **에이전트 런타임**: **self-hosted Hermes (Nous Research)** 를 우리 VPS(Hostinger)에 직접 설치·통제. 관리형 Paperclip 보드 + 관리형 hermes는 **제거.**
- **실행 위치**: **(A) 엔진이 실행** — Hermes는 분석 + *읽기전용 조사*만, 사람 승인 뒤 **엔진이 한정 안전도구 실행.** (신뢰 쌓이면 추후 Hermes 실행(B)로 확장 검토.)

## 2. 타깃 아키텍처 (한 줄)
**우리 소유: [지식 wiki(매장/부서 스코프+권한) + 출력 검증 + 실행 안전(화이트리스트·승인·측정)]. Hermes = 우리가 스코프 컨텍스트를 먹이고, 묶인 도구로 (읽기전용) 조사·분석하는 런타임.**

## 3. 파이프라인 6단계
1. **감지(Detect)** — 기존 RPC/임계값(detection_settings) 유지. 이상치→안건(strategy_executions). evidence(수치) 표준화.
2. **분석(Analyze)** — 엔진이 도메인별 역할 프롬프트로 **self-hosted Hermes** 태스크 트리거. Hermes가 **우리 MCP 지식검색**으로 스코프 지식 회수 → 분석 → 구조화 출력(진단+제안액션+도구태그+정량효과). **출력 검증 게이트**(스키마·수치 정합·환각 차단)로 불량 거부·재시도.
3. **승인(Approve)** — 콘솔에서 운영자가 실행할 액션 선택·승인. 안건 식별 가능(title 표준화).
4. **실행(Execute)** — 승인 액션을 **화이트리스트 안전도구 레지스트리**(가역·멱등·스코프)로 엔진이 실행. before 스냅샷 자동.
5. **측정(Measure)** — 실행 전/후 지표 before→after 정량화(구별된 윈도).
6. **보고·발송(Report & Deliver)** — **단일 신뢰 게이트**(내용 품질·수신자·채널 검증)로 통과 후 등록 채널(이메일/메신저) 발송.

## 4. Hermes ↔ 엔진 연동 계약 (확정 개념)
- Hermes는 **CLI/에이전트 우선, REST API 없음**(문서 확인). 트리거 = VPS SSH로 CLI 호출(비대화).
- **통합 단일 경계 = 우리 MCP 서버.** Hermes를 "우리 MCP 도구 + 최소 안전 내장도구"로만 화이트리스트(`hermes tools`/`hermes mcp`). `execute_code`·위험 도구 차단.
  - MCP 노출 도구: **(1) 스코프 지식검색**(권한·스코프 강제), **(2) 읽기전용 조사**(데이터 조회 등). *실행(쓰기) 도구는 엔진이 직접 — A안.*
- 보안: command-approval ON · 컨테이너 격리(Docker/Singularity) · 구조화 출력(파서/SGLang).
- 출처: hermes-agent.nousresearch.com/docs (mcp, security, cli-commands, configuration).

## 5. 지식 wiki 레이어 (확정)
- **소스 = GitHub `ATOMOS_BRAIN/knowledge/`** (마크다운, 폴더=스코프). VPS 아님(휘발·권한우회 회피).
- **색인 = Supabase pgvector** (임베딩 + 메타: scope/dept/store/read_scope/source/verified). git push → CI/sync 잡으로 색인.
- **검색/주입 = 우리 MCP `knowledge.search(ctx)`** — 호출 역할+안건 기준 스코프·권한 필터 통과분만 반환. Hermes는 지식을 보유 안 함.
- **스코프**: `global` / `brand:{br_id}` / `dept:{SCM|FINANCE|…}` / `store:{st_id}`.
- **권한 2층**: GitHub 협업자(누가 *작성*) + MCP 런타임(각 역할/안건이 *읽을* 네임스페이스).
- **지식 2종**: 참조지식(사람 큐레이션) — **v1 채택**. 학습지식(검증된 사이클 자동축적) — **v2로 연기**.
- **품질 가드**: 학습은 검증된 사이클만 / 사람 "틀림" 피드백 / 출처·신뢰도 메타.
